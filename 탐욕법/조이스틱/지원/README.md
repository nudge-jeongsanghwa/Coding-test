## 문제 풀이

조이스틱을 상하로 움직이는 것(알파벳 움직이기)은 쉽게 풀이가 가능함.

문제에서 어려운 부분은 조이스틱을 좌우로 움직이는 것(커서 움직이기)에 해당함.


## Tip
기본적으로 커서를 0번째 index부터 시작하여 n-1번째 index로 이동하는 과정에서 n-1번 커서를 움직이게 되는데 이보다 더 적게 움직일 수 있는 경우를 찾아야 함.

예를 들어, 
B A A B 와 같은 케이스는 B -> A -> A -> B가 아닌 B -> B가 훨씬 적게 움직일 수 있음.

또한, 시작 위치가 0번째 index이기 때문에 이를 기준으로 앞뒤로 순회해야 할 알파벳의 개수에 따라 1) 앞으로 갔다가 뒤로가기 2) 뒤로 갔다가 앞으로 오기도 움직이는 횟수가 다르기 때문에 이 부분도 고려해야 함.

1) move1 = i * 2 + (n - next_i)
2) move2 = (n - next_i) * 2 + i

이를 `min`을 이용하여 계속 업데이트 하면 정답을 구할 수 있음.

각 케이스에서 앞 -> 뒤, 뒤 -> 앞 등등에 대한 케이스를 순회하며 더 최적의 해를 찾는 부분 때문에 그리디로 분류되었나 싶긴 하지만 오히려 탐색으로 분류되는 것이 더 적합하지 않나 싶음.
